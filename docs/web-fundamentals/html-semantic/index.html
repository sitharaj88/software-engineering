HTML & Semantic Markup

HTML (HyperText Markup Language) is the foundational language of the web. It provides the structure and meaning of web content. While CSS handles presentation and JavaScript handles behavior, HTML defines what the content is â€” and semantic HTML tells both browsers and assistive technologies why it matters.

Why Semantic HTML Matters

Semantic HTML uses elements that convey meaning about the content they contain, rather than just its appearance. Compare these two approaches:

  • Non-Semantic (Bad)
  • Semantic (Good)

The semantic version provides three critical benefits:

  1. Accessibility â€” Screen readers can navigate by landmarks (header, nav, main, footer)
  2. SEO â€” Search engines better understand content structure and importance
  3. Maintainability â€” Developers instantly understand the purpose of each section

Choosing the Right Element

Decision Tree for Sections

Ask yourself these questions in order:

  1. Is it the main content? Use <main> (only one per page).
  2. Could it stand alone as a separate document (RSS feed, syndication)? Use <article>.
  3. Is it a thematic group with a heading? Use <section>.
  4. Is it tangentially related content? Use <aside>.
  5. Is it purely for styling/layout with no semantic meaning? Use <div>.

Document Outline and Headings

HTML headings (<h1> through <h6>) create an implicit document outline. Proper heading hierarchy is essential for both accessibility and SEO.

Common Heading Mistakes

  • Do not skip heading levels (e.g., jumping from h1 to h3)
  • Do not use headings for styling purposes â€” use CSS instead
  • Each page should have exactly one h1
  • Screen reader users often navigate by headings, so a clear hierarchy is critical
