Regex & Text Processing

Regular expressions (regex or regexp) are patterns used to match character combinations in strings. They are one of the most powerful tools in a programmerâ€™s toolkit â€” used for validation, search-and-replace, data extraction, log analysis, and much more. This section covers regex fundamentals through advanced patterns, plus essential Unix text processing tools.

What Are Regular Expressions?

A regular expression is a sequence of characters that defines a search pattern. At its simplest, a regex is just a literal string that matches itself. At its most complex, it can describe intricate patterns with repetition, alternation, grouping, and context-sensitive matching.

Regex Engines: NFA vs DFA

Understanding regex engine types helps you predict performance and behavior.

NFA (Nondeterministic Finite Automaton)

Most programming languages (Python, JavaScript, Java, .NET, Ruby, Perl) use NFA engines:

  • Backtracking â€” tries one path, backtracks on failure
  • Supports all features: backreferences, lookahead, lookbehind, lazy quantifiers
  • Performance â€” can be exponential in pathological cases (catastrophic backtracking)
  • Left-to-right, leftmost match â€” returns the first match found

DFA (Deterministic Finite Automaton)

Some tools (grep, awk, lex) use DFA or hybrid engines:

  • No backtracking â€” examines each character exactly once
  • Performance â€” guaranteed linear time O(n)
  • Limited features â€” no backreferences, no lookahead/lookbehind
  • Leftmost longest match â€” returns the longest possible match
