Greedy vs Lazy Quantifiers

By default, quantifiers are greedy â€” they match as much as possible. Adding ? makes them lazy â€” they match as little as possible.

GreedyLazyMeaning
**?0 or more (as few as possible)
++?1 or more (as few as possible)
???0 or 1 (prefer 0)
{n,m}{n,m}?Between n and m (as few as possible)
  • Python
  • JavaScript

Prefer Negated Classes Over Lazy Quantifiers

Instead of <.*?>, use <[^>]*>. The negated character class avoids backtracking entirely, making it faster and more predictable. This is a general principle: whenever you know the terminating character, use a negated class.

Pattern Testing and Debugging

When building complex patterns, use these strategies:

  1. Start simple â€” Build your regex incrementally, testing each part
  2. Use a regex tester â€” Tools like regex101.com explain each component
  3. Test edge cases â€” Empty strings, special characters, very long input
  4. Check for catastrophic backtracking â€” Nested quantifiers can cause exponential time
  5. Add comments â€” Use verbose mode (x flag) for complex patterns
